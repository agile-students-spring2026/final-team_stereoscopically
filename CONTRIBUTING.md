# Contributing Guide 

This document explains how contributors can work on this project. 

## Team Norms 
- Communicate respectfully with team members. 
- Attend sprint meetings. 
- Complete assigned tasks on time. 
- Help review teammates' work when needed. 

## Git Workflow 

We use GitHub for collaboration. 

- Do not push directly to the main branch. 
- Create a new branch for your work. 
- Commit and push your changes. 
- Open a Pull Request before merging to main. 

## Contributing Rules 

- Use separate branches for new features or fixes. 
- Write clear commit messages. 
- All changes should go through a Pull Request. 

## Local Setup 

1. Clone the repository 
`git clone <repository-url>`

2. Go into the project folder 
`cd project-name`

More setup and testing instructions will be added in later sprints. 

## Coding Standards 

- Write clear and readable code. 
- Use meaningful variable and function names. 
- Keep the code consistent with the existing project structure. 
- Comment code when necessary to explain complex logic. 

## Git Workflow 

We use GitHub for collaboration.

- Do not push directly to the main branch. 
- Create a new branch for your work. 
- Commit and push your changes. 
- Open a Pull Request before merging to main. 
- At least one teammate should review the Pull Request before merging.
